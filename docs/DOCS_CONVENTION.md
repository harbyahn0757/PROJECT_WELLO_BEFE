## 1. 공통 헤더 형식 (메타데이터)

모든 마크다운 문서는 **제목 바로 아래**에 다음 세 항목을 반드시 포함합니다.

```markdown
# 문서 제목

**생성일**: YYYY-MM-DD  
**작업일자**: YYYY-MM-DD  
**작업내용**: (한 줄 요약, 50자 내외 권장)

---
```

| 항목 | 설명 |
|------|------|
| **생성일** | 문서가 최초로 만들어진 날. 알 수 없으면 `미상` 또는 폴더 날짜/추정일 사용. |
| **작업일자** | 해당 작업을 수행한 날. 날짜 폴더(`YYYY-MM-DD_작업명`) 안에 있으면 폴더 날짜와 동일하게. |
| **작업내용** | 문서의 목적을 한 줄로 요약 (50자 내외 권장). |

- 기존에만 있는 항목(작업자, 최종 업데이트 등)은 유지해도 됨.
- **생성일 / 작업일자 / 작업내용** 세 가지는 모든 .md에 추가하는 것을 원칙으로 함.

---

## 2. 폴더 구조

| 구분 | 경로 | 용도 |
|------|------|------|
| 날짜별 작업 내역 | `docs/YYYY-MM-DD_작업명/` | 특정 일자에 수행한 작업 문서 |
| 참조/가이드 | `docs/참조/` (하위: 운영_가이드, 기술, 파트너 등) | 날짜와 무관한 상시 참조 문서 |
| 주제별 | `docs/partner/`, `docs/prompt_engineering_refactoring/`, `docs/scripts/` | 주제별 문서 (내부 .md에도 메타데이터 적용) |

### 날짜 폴더 명명 규칙

- 형식: `YYYY-MM-DD_작업명` (예: `2026-01-10_배포_가이드`)
- 작업명은 한글 또는 영문, 밑줄로 단어 구분

### 참조 문서 하위 분류 예시

- `참조/운영_가이드/` — 데이터 삭제, 배포, 클라이언트 정리 등
- `참조/기술/` — RAG, FLOW, DB 분석, 비밀번호 설계, 블록체인 등
- `참조/파트너/` — 파트너 통합 매뉴얼, 통합 보고서 등

---

## 3. 새 문서 추가 시

1. **작업 내역 문서** (특정 일자 작업): `docs/YYYY-MM-DD_작업명/` 폴더를 만들거나 기존 폴더에 추가.
2. **참조 문서** (상시 가이드): `docs/참조/` 하위 적절한 폴더에 추가.
3. 모든 새 .md 파일에는 상단에 **생성일, 작업일자, 작업내용**을 반드시 기입.
4. 문서 추가·이동 시 `docs/INDEX.md`를 함께 갱신.

---

## 4. 파일 명명 규칙

- 마크다운: `UPPERCASE_WITH_UNDERSCORES.md` 또는 `kebab-case.md`, 한글 파일명 가능
- 날짜 폴더 내 파일명에 날짜를 중복 넣지 않아도 됨 (폴더가 날짜를 나타냄)
