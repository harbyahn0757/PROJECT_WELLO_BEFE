## 🎨 색상 시스템 정책

### 왜 하드코딩을 금지하는가?

#### 1. **일관성 유지**
```scss
// ❌ 나쁜 예: 하드코딩
.card {
  background: #FEF9EE;
  color: #A16A51;
  border: 1px solid #EAE4D7;
}

// ✅ 좋은 예: 토큰 사용
.card {
  background: $background-cream;
  color: $brand-brown-dark;
  border: 1px solid $border-beige;
}
```

**이유:**
- 같은 색상을 여러 곳에서 사용할 때 일관성 보장
- 한 곳에서 색상 변경 시 전체에 자동 반영
- 디자인 시스템의 단일 진실 공급원(Single Source of Truth)

#### 2. **유지보수성 향상**
```scss
// ❌ 문제: 색상 변경 시 모든 파일 수정 필요
// MainPage.scss: background: #FEF9EE;
// HealthTrendsHeader.scss: background: #FEF9EE;
// TrendsSection.scss: background: #FEF9EE;
// ... 10개 파일 수정 필요

// ✅ 해결: 한 곳만 수정
// _variables.scss: $background-cream: #FEF9EE; → #F5F0E8;
// 모든 파일에 자동 반영
```

#### 3. **디자인 토큰 시스템**
```scss
// _variables.scss - 모든 색상 정의
$background-cream: #FEF9EE;              // 배경색
$brand-brown-dark: #A16A51;               // 진한 브라운
$border-beige: #EAE4D7;                  // 경계선

// 사용 시
@use '../../../styles/variables' as *;

.component {
  background: $background-cream;          // ✅ 토큰 사용
  color: $brand-brown-dark;               // ✅ 토큰 사용
}
```

#### 4. **테마 변경 지원**
```scss
// 다크 모드 지원 시
$background-cream: #FEF9EE;              // 라이트 모드
$background-cream-dark: #2D2D2D;         // 다크 모드

// 한 줄로 테마 전환 가능
$background-cream: $background-cream-dark;
```

#### 5. **접근성 보장**
```scss
// 색상 대비 비율 자동 계산
$brand-brown-dark: #A16A51;              // WCAG AA 기준 충족
$background-cream: #FEF9EE;               // 대비 비율 4.5:1 이상

// 하드코딩 시 대비 비율 확인 어려움
```

## 📋 색상 시스템 구조

### 배경색 계층
```
$background-cream (#FEF9EE)
  ├── 전체 페이지 배경
  ├── 헤더 배경
  └── 탭 활성 배경

$overlay-strong (rgba(255, 255, 255, 0.9))
  └── 카드 배경
```

### 브라운 계열 계층
```
$brand-brown (#7c746a)
  └── 기본 브라운 (닷 인디케이터)

$brand-brown-dark (#A16A51)
  ├── 탭 비활성 배경
  ├── 측정 버튼 배경
  └── 강조 텍스트

$brand-brown-darker (#55433B)
  ├── 탭 활성 텍스트
  └── 강조선

$brand-brown-hover-dark (#8B5A47)
  └── 탭 호버 상태
```

### 회색 계열 계층
```
$gray-800 (#2d3748)
  └── 카드 제목

$gray-600 (#718096)
  └── 본문 텍스트

$gray-550 (#737373)
  └── 헤더 업데이트 정보

$gray-450 (#565656)
  └── 보조 텍스트

$gray-300 (#e2e8f0)
  └── 닷 인디케이터 비활성
```

### 경계선 색상
```
$border-beige (#EAE4D7)
  ├── 탭 하단 경계선
  ├── 헤더 구분선
  └── 카드 측정일 구분선
```

## 🔄 간격 시스템 구조

### 8px 기준 그리드 시스템
```
$spacing-xs: 8px    → 요소 내부 작은 간격
$spacing-sm: 12px   → 요소 간 작은 간격
$spacing-md: 16px   → 요소 간 기본 간격
$spacing-lg: 24px   → 섹션 간격, 카드 패딩
$spacing-xl: 32px   → 큰 섹션 간격
```

### 사용 예시
```scss
// 헤더
padding: $spacing-sm $spacing-md;        // 12px 16px

// 카드
padding: $spacing-lg;                    // 24px

// 카드 간격
gap: $spacing-md;                        // 16px

// 섹션 간격
margin-bottom: $spacing-lg;              // 24px
```

## 🎯 핵심 원칙

1. **단일 진실 공급원**: 모든 색상/간격은 `_variables.scss`에서만 정의
2. **토큰 우선 사용**: 하드코딩 금지, 토큰 사용 필수
3. **의미 있는 이름**: `$brand-brown-dark` (의미) > `#A16A51` (값)
4. **일관성 보장**: 같은 용도는 같은 토큰 사용
5. **유지보수성**: 변경 시 한 곳만 수정
