## 2. 인풋 UX 패턴 TOP 5 (차용 추천)

### P1. Heart Age Delta 슬라이더 (JBS3 방식)
**근거**: JBS3 — 첫 5개월 140만 뷰. 가장 직관적인 "내가 바꾸면 몇 살 젊어지나" 패턴.

- 단일 슬라이더 1개 조작 → 건강나이/위험도 숫자가 실시간으로 바뀜
- 슬라이더 레이블: 현재값 | 목표값
- 색 변화: 빨강(위험) → 노랑(경계) → 초록(안전) 그라디언트
- **우리 적용**: BMI·금연·금주 슬라이더에 이미 적용됨 → 운동·식습관·수면으로 확장 시 동일 패턴 사용

### P2. 카드 선택형 생활습관 입력 (Noom / Omada 방식)
**근거**: Noom 컬러 코딩 시스템 — 복잡한 영양 과학을 초록/노랑/빨강으로 단순화 → 레스토랑에서 실시간 결정 가능.

- 운동 빈도: [전혀 안 함] [주 1-2회] [주 3-4회] [매일] 카드 선택
- 식습관: [패스트푸드 위주] [보통] [채소 위주] 카드 선택
- 수면: [5시간 미만] [6시간] [7-8시간] [8시간 초과]
- **장점**: 슬라이더 정밀도 없어도 됨 / 모바일 원핸드 가능
- **우리 적용**: 문진 답변 변경 → 결과 재계산 시나리오에 최적

### P3. Before-After 나란히 비교 (Livongo / CVD Risk 방식)
**근거**: ASCVD 계산기 — "이 인자를 바꾸면 위험도 X% 줄어든다" 직접 병렬 표시.

- 왼쪽: 현재 (빨간 배경)
- 오른쪽: 개선 후 (초록 배경)
- 중간: 화살표 + delta 숫자 (예: "-12%p", "-3세")
- **우리 적용**: 현재 상태 vs. 목표 달성 시 16개 질환 위험도 차이 표

### P4. 타임라인 슬라이더 (우리 자산 확장)
**근거**: 우리 α 감쇠 로직이 이미 0/6/12/60개월 지원.

- 하단에 타임라인 바: [지금] → [6개월] → [1년] → [5년]
- 타임라인 이동 시 건강나이 delta 곡선 애니메이션
- "지금 바꾸면 vs. 6개월 후 바꾸면" 비용 차이 시각화
- **차별점**: 경쟁 서비스에 없는 우리만의 강점

### P5. AI 채팅 목표 입력 (Lark Health / Noom AI 방식)
**근거**: Lark — 온보딩 개선 후 2일 내 완료율 +21% / Noom 2024 Welli AI.

- 자연어 입력: "이번 달 3kg 빼고 싶어요"
- AI가 자동으로 슬라이더 값 세팅
- 결과: "목표대로면 6개월 후 심혈관 위험도 8% 감소 예상"
- **우리 적용**: 문진 채팅 완료 후 "이런 목표 어떠세요?" 추천 연동

## 3. 아웃풋 시각화 패턴 TOP 5

### V1. 건강나이 숫자 카운트다운 애니메이션
**근거**: JBS3 Heart Age — "몇 살 젊어지나" 숫자가 가장 강한 동기부여 신호.

- 슬라이더 드래그 시 숫자가 실시간 카운트업/다운
- 큰 폰트 (>60px) + 색 변화 (빨강 → 초록)
- 아래: "지금 금연하면 건강나이 4세 감소"
- **기술**: CSS counter + React state → 부드러운 숫자 보간

### V2. 레이더 차트 변화 오버레이
**근거**: 16개 질환 × 영역별 시각화 — 현재 vs. 개선 후를 같은 레이더에 투명도로 겹쳐 표시.

- 현재: 빨간 음영 (넓을수록 위험)
- 목표: 초록 점선 (겹치는 영역이 개선 효과)
- 호버: 해당 질환명 + delta 수치 팝오버
- **우리 적용**: 기존 ratio/rank 데이터를 레이더로 재구성

### V3. 이중 게이지 (현재 vs. 목표)
**근거**: Livongo 연결기기 실시간 피드백 패턴 → 목표 구간 강조.

- 반원형 게이지: 빨강(위험) - 노랑(경계) - 초록(정상) 색띠
- 현재 바늘 + 목표 바늘 (점선)
- 중앙: 위험도 % 숫자 / 아래: "목표까지 X%p"

### V4. 시간축 예측 라인차트
**근거**: Omada Health — A1C·체중 장기 추적 / 우리 α 감쇠 로직 시각화에 최적.

- X축: 오늘 / 3개월 / 6개월 / 1년 / 5년
- Y축: 질환 위험도 % 또는 건강나이
- 실선: 현재 추이 (변화 없음)
- 점선: 개선 추이 (슬라이더 값 반영)
- 음영: 두 선 사이 gap = 개선 효과 영역

### V5. 등수 변화 배지 (Will Rogers 현상 시각화)
**근거**: 우리 자산 — rank 데이터 이미 있음 / Samsung Health 소셜 비교.

- "같은 40대 남성 중 현재 상위 42% → 개선 후 상위 28%"
- 배지 형태: [현재 등수] → [목표 등수] 화살표
- 컬러: 구리(하위) → 은(중간) → 금(상위) 티어

## 4. 게이미피케이션 요소 TOP 5

### G1. 건강나이 마일스톤 축하 (Confetti)
**근거**: 헬스케어 게이미피케이션 연구 — 디지털 confetti가 현금 인센티브보다 비용 대비 효과적.

- 건강나이 1세 개선 시: confetti 애니메이션 + "1세 젊어졌어요!" 팝업
- 마일스톤: -1세 / -3세 / -5세 각각 다른 이펙트
- Apple Health 활동 링 완성 패턴 벤치마크

### G2. 스트릭 + 습관 연속 기록 (Hinge Health 방식)
**근거**: Hinge Health — 스트릭 카운터로 운동 지속성 향상.

- "3일 연속 운동 기록 유지 중" 배너
- 스트릭 끊기면: "오늘 다시 시작하면 건강나이 회복 가능" 격려 메시지
- **우리 적용**: 문진 응답 → 생활습관 개선 실천 → 스트릭 연동

### G3. 동일 집단 비교 (Samsung Health 소셜 챌린지 방식)
**근거**: Samsung Health Together — 친구 경쟁 + 글로벌 챌린지로 참여율 향상.

- "같은 연령대·성별 상위 20%에 진입하려면 운동 주 2회만 더하면 됩니다"
- 익명 집단 비교 (개인정보 비노출)
- **우리 자산**: rank 데이터 이미 있음 → 즉시 적용 가능

### G4. 목표 달성 시나리오 스토리텔링 (Noom CBT 방식)
**근거**: Noom — 심리 기반 단기 목표 → 장기 습관 연결.

- "지금 금연하면 → 3개월 후 폐기능 회복 → 6개월 후 심혈관 위험 15% 감소 → 1년 후 건강나이 -3세"
- 단계별 내러티브 + 숫자 결합
- **우리 자산**: α 감쇠 + 시간축 이미 구현 → 스토리 레이어만 추가

### G5. 도전 과제 카드 ("이번 주 한 가지 바꿔보기")
**근거**: Omada OmadaSpark + Lark DPP 코칭 — 소규모 목표가 지속성 향상.

- "이번 주: 운동 1회 추가 → 건강나이 0.5세 개선 예상"
- 완료 시 체크마크 + 다음 도전 카드 언락
- 헬스케어 게이미피케이션 시장 2024년 46억 달러 → 2030년 100억 달러 예상 (실증치)

## 5. 기술 구현 권장 패턴

### T1. 클라이언트 사이드 계산 (서버 호출 없음)
**근거**: ASCVD 계산기 — NHS heart age가 JavaScript 클라이언트 계산으로 140만 뷰 처리.

- 위험도 재계산 로직을 **React 컴포넌트 내 순수함수**로 구현
- 슬라이더 변경 → state 업데이트 → 즉시 재계산
- 서버 debounce 불필요 / 네트워크 지연 없음
- **우리 자산**: α 감쇠 수식 이미 Python 구현 → JS 포팅 필요

### T2. debounce 전략 (서버 저장 시)
- 슬라이더 변경: **클라이언트 즉시 계산** (debounce 없음)
- DB 저장: **300ms debounce** (서버 부하 방지)
- 패턴: `useCallback + useDeferredValue` (React 18)

### T3. 애니메이션 — Motion (구 Framer Motion)
**근거**: 2025년 React 애니메이션 1위 라이브러리 / 120fps Web Animations API.

- 숫자 카운트업: `useSpring` + 보간 함수
- 슬라이더 → 차트 연동: `useMotionValue` + `useTransform`
- 레이더 차트 변화: `AnimatePresence` + `layout` 애니메이션
- 설치: `npm install motion` (tree-shakable)

### T4. 차트 라이브러리 — Recharts (기존 스택 유지)
- 기존 프로젝트 Recharts 사용 중 → 교체 불필요
- RadarChart: 현재/목표 데이터셋 2개 + 투명도 조정
- AreaChart: 시간축 예측선 (실선 + 점선 + 음영)
- ComposedChart: 게이지 커스텀 구현

### T5. 성능 최적화
- `React.memo` + `useMemo`로 16개 질환 재계산 최적화
- 슬라이더 드래그 중 heavy calculation: `requestIdleCallback` 큐
- 모바일 저사양 대응: 애니메이션 복잡도 `prefers-reduced-motion` 감지 후 자동 저하

## 6. 우리 엔진 확장 로드맵 (운동/식습관/수면 추가)

### 현재 자산 (확인됨)
- BMI 슬라이더 → 실시간 위험도 재계산
- 금연 시뮬레이션 구현됨
- 금주 시뮬레이션 구현됨
- α 감쇠: 0/6/12/60개월 시간축 지원
- 16개 질환 × ratio/rank 실시간 재계산
- 건강나이 delta 계산
- Will Rogers 현상 (등수 변화) 구현됨

### 추가 필요 변수 및 근거

| 변수 | 근거 데이터 | 구현 난이도 |
|------|------------|------------|
| **운동 빈도** | 헬스케어 게이미피케이션 연구: 운동 앱에서 게이미피케이션 시 효과 유의미 향상 | 낮음 — 카드 선택 + 계수 적용 |
| **식습관 (채소·가공식품 비율)** | Noom 컬러 코딩 — 단순 3단계로도 행동 변화 유도 | 낮음 — 3단계 카드 선택 |
| **수면 시간** | Biological Age 연구: 7-8시간 수면 → 생물학적 나이 감소에 기여 | 낮음 — 슬라이더 또는 카드 |
| **스트레스 수준** | Biological Age Calculator (MiniWebtool) 12요소 중 포함 | 중간 — 주관적 지표 수치화 필요 |
| **약물 복용** | JBS3 — 혈압약·콜레스테롤약 복용 여부가 Heart Age에 영향 | 중간 — 문진 연동 필요 |

### 문진 연동 시나리오 (핵심)
사장님 요청: "문진 답변을 바꾸면 결과가 재미있게 보여줘야 한다"

```
문진 답변 변경 → 변수 매핑 → 위험도 재계산 → 시각화 업데이트

예:
- Q: "주 몇 회 운동하세요?" [0회] → [3회] 변경
  → 심혈관 위험도 -8% / 건강나이 -1.5세 즉시 표시
  
- Q: "흡연 여부" [현재 흡연] → [비흡연] 변경
  → 폐암 위험도 -30% / 건강나이 -4세 즉시 표시
```

## 7. "건강 플래너" Phase 1-3 제안

### Phase 1 — 슬라이더 확장 (4주)
**목표**: 운동·식습관·수면 슬라이더 추가 + 실시간 재계산

- [ ] 운동 빈도 카드 선택 (0/1-2/3-4/5회+) → 16개 질환 계수 적용
- [ ] 식습관 3단계 카드 (불량/보통/양호) → BMI·대사 관련 질환 계수 적용
- [ ] 수면 슬라이더 (5h 미만~9h 이상) → 심혈관·인지 관련 계수 적용
- [ ] 건강나이 delta 표시에 신규 변수 반영
- [ ] 카운트업/다운 애니메이션 (Motion 라이브러리)

**KPI**: 슬라이더 조작 세션당 평균 횟수 > 3회

### Phase 2 — Before-After 비교 + 타임라인 (4주)
**목표**: 시각적 임팩트 강화 + 시간축 스토리텔링

- [ ] Before(현재) / After(목표) 나란히 비교 카드
- [ ] 시간축 슬라이더 (6개월/1년/5년) → 예측 라인차트
- [ ] 레이더 차트 오버레이 (현재 vs. 목표 16개 질환)
- [ ] "지금 바꾸면 vs. 1년 후 바꾸면" delta 비용 표시
- [ ] 공유 버튼 ("내 건강 플랜 공유하기")

**KPI**: 공유 전환율 > 5% / 리텐션 D7 > 40%

### Phase 3 — 게이미피케이션 + AI 추천 (4주)
**목표**: 동기부여 강화 + 습관 형성 루프

- [ ] 건강나이 마일스톤 confetti 애니메이션
- [ ] 동일 집단 비교 배지 (rank 데이터 활용)
- [ ] 주간 도전 과제 카드 ("이번 주 한 가지")
- [ ] AI 목표 추천 ("운동 2회 더하면 건강나이 -1.5세")
- [ ] 문진 답변 ↔ 슬라이더 양방향 연동

**KPI**: 주간 활성 사용자 WAU / 도전 완료율 > 30%
