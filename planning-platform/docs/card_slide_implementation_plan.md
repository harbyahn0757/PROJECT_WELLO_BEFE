# 카드 슬라이드 구현 계획

## 목표 구조

### 카드 순서 (총 4개)
1. **카드 0 (기본)**: 실제 프로세스 단계 리스트 (처음부터 표시)
2. **카드 1**: 문진 반영 내용 (STEP 1 결과 도착 후)
3. **카드 2**: 선택하신 항목 분석 (카드 1 타이핑 완료 후)
4. **카드 3**: 실제 프로세스 단계 카드 (모든 작업 완료 후)

### 슬라이드 동작
```
[카드 0] → [카드 1] → [카드 2] → [카드 3]
  ↓          ↓          ↓          ↓
왼쪽으로    왼쪽으로   왼쪽으로   최종 표시
밀려남      밀려남     밀려남
```

## 구현 단계

### 1. 카드 상태 관리
- `currentCardIndex`: 현재 표시 중인 카드 인덱스 (0, 1, 2, 3)
- `card0Visible`: 카드 0 표시 여부 (기본 true)
- `card1Visible`: 카드 1 표시 여부 (STEP 1 결과 도착 후)
- `card2Visible`: 카드 2 표시 여부 (카드 1 완료 후)
- `card3Visible`: 카드 3 표시 여부 (모든 작업 완료 후)

### 2. 카드 전환 로직
```typescript
// 카드 0 → 카드 1
- STEP 1 결과 도착 + 딜레이(5초 이상) → 카드 0 슬라이드 아웃, 카드 1 슬라이드 인

// 카드 1 → 카드 2
- 카드 1 타이핑 완료 → 카드 1 슬라이드 아웃, 카드 2 슬라이드 인

// 카드 2 → 카드 3
- 모든 작업 완료 (stage === 'saving' 완료) → 카드 2 슬라이드 아웃, 카드 3 슬라이드 인
```

### 3. 슬라이드 애니메이션
- **슬라이드 아웃**: `transform: translateX(-100%)` + `opacity: 0`
- **슬라이드 인**: `transform: translateX(0)` + `opacity: 1`
- **초기 위치**: `transform: translateX(100%)` (오른쪽에서 시작)

### 4. 카드 내용

#### 카드 0 (기본 프로세스 단계)
- 현재 `processing-modal__details` 영역 내용
- 단계별 진행 상황 표시 (✓, ⟳, ○)
- 예상 시간 표시

#### 카드 1 (문진 반영 내용)
- `step1Result.survey_reflection`
- 타이핑 효과 적용
- 하이라이트 처리

#### 카드 2 (선택하신 항목 분석)
- `step1Result.selected_concerns_analysis`
- 타이핑 효과 적용
- 하이라이트 처리

#### 카드 3 (프로세스 단계 카드)
- 카드 0과 동일한 내용이지만 카드 형태로
- 최종 완료 상태 표시
- 모든 단계 완료 표시

### 5. 타이핑 효과
- 카드 1, 2에만 타이핑 효과 적용
- 카드 0, 3은 즉시 표시

### 6. 딜레이 설정
- 첫 번째 카드(카드 1) 표시 전 딜레이: `FIRST_CARD_DELAY` (5초 이상)
- 카드 전환 딜레이: 800ms (슬라이드 애니메이션 시간)
