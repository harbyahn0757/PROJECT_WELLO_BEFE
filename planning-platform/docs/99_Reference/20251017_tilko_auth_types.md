# 틸코 인증 방식 분석

## 🔐 지원되는 인증 방식

| PrivateAuthType | 인증 방식 | 앱/서비스 | 특징 |
|-----------------|-----------|-----------|------|
| **0** | 카카오톡 | KakaoTalk | 현재 사용 중, 가장 범용적 |
| **1** | 페이코 | NHN Payco | 온라인 결제 서비스 |
| **2** | 국민은행모바일 | KB Mobile | 은행 앱 기반 |
| **3** | 삼성패스 | Samsung Pass | 삼성 생체인증 |
| **4** | 통신사Pass | SKT/KT/LG U+ | 통신사 인증 |
| **5** | 신한 | 신한은행 | 은행 앱 기반 |
| **6** | 네이버 | Naver | 네이버 계정 |

## 📱 현재 구현 상태

- **현재**: `PrivateAuthType: "0"` (카카오톡) 하드코딩
- **개선 필요**: 사용자가 인증 방식을 선택할 수 있도록 UI 제공

## 🎯 구현 방안

### 1. 프론트엔드 인증 방식 선택 UI
```typescript
const AUTH_TYPES = [
  { value: '0', label: '카카오톡', icon: 'kakao' },
  { value: '1', label: '페이코', icon: 'payco' },
  { value: '2', label: '국민은행', icon: 'kb' },
  { value: '3', label: '삼성패스', icon: 'samsung' },
  { value: '4', label: '통신사Pass', icon: 'telecom' },
  { value: '5', label: '신한은행', icon: 'shinhan' },
  { value: '6', label: '네이버', icon: 'naver' }
];
```

### 2. 백엔드 동적 처리
- 현재 하드코딩된 `"0"` 값을 프론트엔드에서 받은 값으로 대체
- 각 인증 방식별 특화 로직 추가 (필요시)

### 3. 사용자 경험 고려사항
- **기본값**: 카카오톡 (가장 범용적)
- **인기순 정렬**: 카카오톡 → 네이버 → 페이코 → 통신사Pass → 은행앱들
- **앱 설치 여부 체크**: 선택한 인증 앱이 설치되어 있는지 확인

## 🔍 테스트 방안

1. **카카오톡 (0)**: 현재 정상 작동 확인됨
2. **다른 인증 방식**: 실제 앱이 설치된 환경에서 테스트 필요
3. **오류 처리**: 앱 미설치 시 적절한 안내 메시지

## 📋 구현 우선순위

1. **1단계**: 프론트엔드 선택 UI 추가
2. **2단계**: 백엔드 동적 처리 구현
3. **3단계**: 각 인증 방식별 테스트
4. **4단계**: 오류 처리 및 사용자 안내 개선
