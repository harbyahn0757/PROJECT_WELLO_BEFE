## 2. UX 패턴 분류

### 패턴 A. 퀴즈형 (Quiz-driven)

**대표**: Care/of, 필리, LemonBox, iHerb Wellness Quiz

**특징**:
- 온보딩이 퀴즈 자체. 5~15분 내 완결.
- 문항마다 "왜 이걸 묻는지" 컨텍스트 제공 (Care/of가 특히 탁월).
- 이름 직접 호명, 아이코노그래피, 타이포그래피로 퀴즈를 "재미있는 대화"로 만듦.
- 결과: 즉시 추천 리스트 + 구매 유도.

**강점**: 진입 장벽 낮음, 즉시 몰입, 전환율 높음.
**약점**: 입력 데이터 얕음 (설문 의존), 정확도 한계.

---

### 패턴 B. 대화형 (Conversational AI)

**대표**: Persona Nutrition (영양사 채팅), 모노랩스 IAM (약사 상담), 카카오헬스케어 케어챗

**특징**:
- AI 챗봇 또는 전문가 실시간 상담.
- 상담 기반 정교화 → 초기 설문보다 맥락 풍부.
- 약물 상호작용, 기저질환 등 민감 정보 수집 용이.

**강점**: 신뢰감, 안전망(전문가 검토), 복잡 케이스 처리.
**약점**: 속도 느림, 운영비 높음, 확장성 제한.

---

### 패턴 C. 대시보드형 (Data Dashboard)

**대표**: Rootine, Viome, 필라이즈

**특징**:
- 검사(혈액/DNA/마이크로바이옴) 또는 건강검진 데이터 연동.
- 앱 내 지속적 추적 대시보드 (건강 변화 그래프, 수치 트렌드).
- 영양제 추천이 "데이터 기반 처방"처럼 제시.

**강점**: 정확도 높음, 재방문 유인 강함, "과학적" 신뢰감.
**약점**: 진입 장벽 높음(검사 키트, 비용), 온보딩 복잡.

---

### 패턴 D. 매거진형 (Content-led)

**대표**: iHerb 블로그 + 퀴즈, 뉴트리원 공식몰

**특징**:
- 콘텐츠(아티클, 성분 설명)가 먼저 → 제품 추천이 자연스럽게 노출.
- 트렌드·시즌 기반 큐레이션 ("봄철 추천", "수험생 영양제").
- 추천보다 발견(discovery) 경험.

**강점**: SEO 강함, 신뢰 콘텐츠, 비구매자도 체류.
**약점**: 전환 경로 길고 개인화 약함.

## 4. 우리 경쟁 우위 — 검진 데이터 + RAG

### 4-1. 입력 데이터 비교

|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| Care/of | O | - | - | - | - | - | - |
| Persona Nutrition | O | - | - | - | - | - | O (4,000+) |
| Viome | O | - | - | O | - | - | - |
| Rootine | O | O | O | - | - | - | - |
| 필라이즈 | O | O(연동) | - | - | O(건강검진 10년) | - | - |
| **피어나인 mediArc** | **O** | **O (실측)** | **-** | **-** | **O (실제 검진결과 13종+)** | **O (16개 질환)** | **O (357종 RAG)** |

**결론**: 피어나인은 국내 서비스 중 필라이즈 수준의 실측 검진 데이터를 보유하면서, 해외 Persona 수준의 약물 상호작용 RAG까지 갖춘 유일한 위치. 여기에 16개 질환 위험도 엔진이 추가되어 "단순 영양소 부족" 아닌 "질환 위험도 기반 처방" 수준의 추천이 가능하다.

### 4-2. 현재 한계 (솔직한 진단)

| 한계 | 현황 |
|---|---|
| 추천 종류 | 하드코딩 6종으로 고정 |
| UX | 리포트 내 텍스트 나열 수준, "재미" 요소 없음 |
| 개인화 내러티브 | "OOO님의 혈당 기반" 스토리텔링 부재 |
| 섭취 가이드 | 타이밍/용량/주의사항 안내 없음 |
| 에비던스 노출 | 식약처 기능성 연결 없음 |

## 5. 고도화 방향 제안 — 3가지 시나리오

### 시나리오 A. "스마트 퀴즈" (Quick Win, 2~4주)

**컨셉**: 기존 검진 데이터를 가지고 있으면서도, 처음 사용자에게는 Care/of식 퀴즈 온보딩을 붙임. 검진 데이터가 있으면 퀴즈를 자동 스킵하고 바로 결과 제시.

**핵심 변경**:
- RAG 6종 → 식약처 인정 원료 기반 20~30종으로 확대
- 추천 카드에 "식약처 인정 기능성 + 이유 한 줄" 추가
- "OOO님의 혈압이 높네요 → 마그네슘이 도움될 수 있습니다" 내러티브 1~2줄

**재미 요소**: 검진 수치별 짧은 스토리텔링, 성분 근거 카드

**기대 효과**: 구현 빠르고 리포트 체류 시간 즉시 상승

**위험**: 추천 폭이 좁으면 금방 식음

---

### 시나리오 B. "개인화 리포트 + 약물 경고" (핵심 차별화, 4~8주)

**컨셉**: 기존 RAG 357종 약물 상호작용 + 16개 질환 위험도를 완전히 활용. 복용 중인 처방약을 입력하면 즉시 상호작용 경고 + 안전한 대안 추천. 이 기능은 국내 어떤 서비스도 실측 검진 + 약물 상호작용 조합을 제공하지 않음.

**핵심 변경**:
- 복용 약물 입력창 추가 (문진 or 건강검진 결과 페이지)
- RAG 357종 약물 상호작용 → 경고 배너 ("이 영양제는 A 약과 함께 복용하면 B 효과 감소")
- 추천 결과를 "안전 추천 / 주의 추천 / 제외" 3단계로 분류

**재미 요소**: 약물 경고 배너(안전감), 16개 질환 위험도 개선 시뮬레이션 ("오메가3 복용 시 이상지질혈증 위험 X% 개선 예상")

**기대 효과**: 병원 판매 차별화 포인트, 수검자 신뢰도 급상승

**위험**: 의학적 소견 경계 관리 필요 (마케팅 규칙 준수)

---

### 시나리오 C. "건기식 구독 커머스" (장기 수익화, 2~3개월+)

**컨셉**: 추천 → 즉시 구매 → 정기구독 팩. 필리/모노랩스 모델 + 우리의 검진 데이터 정확도.

**핵심 변경**:
- 추천 결과에 "지금 구매" 버튼 (파트너 커머스 또는 자체 입점)
- 월 구독 팩 ("OOO님 맞춤 3종 팩, 월 39,000원")
- 섭취 알림 (카카오 알림톡 기존 인프라 활용)
- 섭취 체크인 → 위험도 재산정 → "3개월 후 위험도 변화" 리포트

**재미 요소**: 섭취 스트릭(Streak), 위험도 개선 트래킹, "3개월 전 대비 LDL X% 개선" 성과 공유

**기대 효과**: 월 구독 수익, 수검자 재방문, 병원 파트너 수익 공유 모델

**위험**: 커머스 운영 비용, 재고 관리, 제휴 협상 필요

---

## 6. 권장 시나리오 + 구현 로드맵

### 권장: B 우선 → A 병행 → C 검토

"재미있게 풀어내야 한다"는 사장님 지시의 핵심은 경쟁사가 없는 차별화다. 단순 퀴즈(A)는 Care/of가 이미 정점을 찍었고, 구독 커머스(C)는 필리/모노랩스가 선점했다. 우리만이 가진 것은 **실측 건강검진 × 16개 질환 위험도 × RAG 약물 상호작용**이다. 이것이 "재미있게" 들어올 때 차별화가 생긴다.

### Phase별 로드맵

```
Phase 1 (2~4주) — 기반 정비 + 빠른 UX 개선
  목표: 기존 6종 → 20~30종 확대, 스토리텔링 1~2줄 추가
  작업:
    - RAG 건기식 청크 1,683개 중 '식약처 인정 기능성' 필터링 → 추천 후보군 확정
    - 추천 카드 UI 개편: 성분명 + 기능성 요약 + "OOO님 혈압 기반 추천" 레이블
    - 검진 수치별 내러티브 템플릿 10~15종 작성
  KPI: 리포트 내 건기식 섹션 체류 시간, 클릭률

Phase 2 (4~8주) — 약물 상호작용 + 질환 연동
  목표: 국내 유일 "실측 검진 + 약물 경고" 기능 출시
  작업:
    - 복용 약물 입력 UI (문진 연동 or 별도 입력창)
    - RAG 357종 약물 상호작용 → 추천 필터링 + 경고 배너
    - 16개 질환 위험도 → "이 영양소가 도움될 수 있는 이유" 연결
    - 안전/주의/제외 3단계 추천 분류 UI
    - 에비던스: 식약처 기능성 원료 코드 연결
  KPI: 약물 경고 노출률, 추천 변경률, B2B 병원 어필 포인트

Phase 3 (2개월+) — 구독·커머스 가능성 검토
  목표: 수익화 모델 타당성 검증
  작업:
    - 파트너 커머스 제휴 또는 자체 입점 협상
    - 섭취 알림 (카카오 알림톡 기존 인프라)
    - 섭취 체크인 → 위험도 재산정 파이프라인
  선결 조건: Phase 2 CTR 및 사용자 반응 확인 후 진행 결정
```
